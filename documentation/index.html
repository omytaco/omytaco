Getting Started & Installation

Foodster includes the following elements and assets that will help you build your own website: a flexible, dynamic one-page site, useful JavaScript plugins, compiled CSS & LESS files, and a selection of images that are free for commercial use.

File Structure

We've organized the files in a logical way to make your usage of the theme easier.

The index.html file is the only HTML file, and includes all the layout, structure and content for the theme.

The assets folder contains the JavaScript, CSS/LESS, PHP and images for the theme. Keep everything that isn't layout/content related in there.

  • images contains the image files used in the theme.
  • fonts contains the custom font files used in the theme.
  • bootstrap contains the Bootstrap framework CSS, JS and font files. These are the original untouched framework files and should not be edited, removed or changed.
  • js contains the JavaScript plugins and theme JavaScript.
  • css contains the compiled theme CSS and the custom.css file where you can add your own CSS.
  • less contains the LESS files, if you want to use LESS to write/compile your CSS. Custom LESS should be added to the custom.less file.
  • sass contains the SCSS files, if you want to use SASS to write/compile your CSS. Custom SCSS should be added to the custom.scss file.
  • php contains the booking.php script that is required for the 'Book a Table' form to work correctly.

The psd folder contains the Photoshop file for the design of the theme. This is a full, layered Photoshop file that you can edit and use for commercial use.

The documentation folder is where you are now! It contains all the files relating to the documentation, and is not necessary for the theme to work.

Editing the Index.html file

You can use a text editing program, such as Notepad++ (Windows) or Sublime Text (Mac + Windows), to start editing the index.html file. The index.html is the only HTML file that the theme needs, because it is a one page site. The index file contains the HTML markup which creates the structure of the page, as well as the content. Changing the Logo

The logo is included both in the top section of the page, and also in the navigation. Both of these have been included using transparent PNGs. You can either replace them with your own transparent PNG, or replace them with text.

Changing the image source

Images are included on webpages using the <img> HTML element. The src attribute defines where the image file is located.

The default image is stored at assets/images/ and is named logo.png:

<img src="assets/images/logo.png" alt="">

You can either replace the logo.png file with your own, or point the <img> src to a different file:

<img src="assets/images/myawesomelogo.png" alt="">

Replace the image-based navigation logo with text

Remove the highlighted line shown below from the HTML:

<a class="navbar-brand page-scroll" href="#header">
    <img src="assets/images/foodster_navigation_logo.png" alt="Foodster">
</a>

Replace it with:

<span>Your Website Name</span>

If replacing the logo image, check your file permissions

If a file's permissions are incorrect, a brower request for a file, such as logo.png, might return not found (404). If the file exists but you are still receiving a 404 error, it is worth checking the permissions of the image file.

Read how to check and change file permissions (Windows)
Read how to check and change file permissions (Linux + Mac)

Image src paths are most commonly case sensitive

For example, if your file is called MyAwesomeLogo.PNG, the image src attribute should look like this:

<img src="assets/images/MyAwesomeLogo.PNG" alt="" />

The following would be incorrect:

<img src="assets/images/myawesomelogo.png" alt="" />

Using Retina Images

By default, the Foodster theme logo used in the header and navigation bar are retina ready. The theme uses native webfonts and webfont iconography - both vector-based and therefore retina ready as well.

Why Retina?

Many modern devices use high resolution displays, or 'retina' displays. This means that the ppi (pixels per inch) is higher. Essentially, there are more pixels on the screen - so your images need to allow for this if you want them to look sharp on a high resolution display.

Simple Rule: Double the Size

The simplest method is to save any exported graphics double their native size. If your vector logo is 200px wide, export a 400px wide version and append the filename with '@2x'. Retina images can be JPG or PNG format.

Avoid Bitmap, Use Vector instead

Bitmap elements, unless originally exported at 200% size, are unsuitable for use as retina images as they do not keep their quality when scaled.

Retina.JS

The included retina.js JavaScript allows browsers to detect and use retina images when they are needed, your retina images won't be loaded without it.

Retina.JS will not work in your local environment

Please upload the theme to a remote server for the best experience. Retina.JS will not work unless runnning on a server.

Editing the Main Slider

The top of the page includes a full-screen image slider, which uses Bootstrap's Carousel JavaScript plugin. This makes it easy to change the images and settings used.

For full instructions of how the Carousel plugin works, see the Bootstrap documentation.

Structure & setup

The data attribute data-ride and its associated property carousel are required for the slider to work.

Each slide is contained within a div element with class .item.

The slide with class .active is the first to appear. Only one slide should have this class.

Slide content is absolutely positioned to the center (top/left) of the page, the CSS properties for this are applied to the div elements with class .slide-content.

The classes animated, fadeInDown, fadeInDown and zoomIn are used for the CSS animations. If you do not want animation, remove these classes from the .slider-content element.

By default the slider uses the Bootstrap left and right Carousel arrow controls, allowing users to navigate between slides. The slider will work without these controls, and the glyphicons used for the arrows can also be changed to FontAwesome icons, or to image-based icons.

To change the images used in the slider, change the background-image url src attribute to point to your own images.

Using LESS

LESS is a stylesheet language that you can use to extend the ability of CSS. It can do all sorts of clever things, using features like variables, nesting, mixins, operators and functions. In essence, LESS brings features commonly seen in other programming languages to CSS. Our themes always come with LESS files that you can use to quickly change fonts, colors and other variables.

Getting started with LESS

To get started with LESS, you'll need a text editor to edit the .less files, and you'll need an app to compile the .less files into ordinary CSS files. Web browers cannot interpret LESS files, so simply including a .less file like you would with a CSS file won't work. A compiler interprets the LESS code and creates an ordinary CSS file from it.

We recommend SimpLESS as a free solution for Mac/Windows users, or Mixture.io as a paid-for solution for Mac/Windows.

Editing LESS files

Once you've got a compiler installed and runnning on your project, you can edit the LESS files. If your setup is working correctly, as soon as you make any changes to a file and save it, the compiler should recompile your CSS files automatically. You can edit your LESS files just the same as you would normally do with CSS - and you can use ordinary CSS without any features of LESS inside a .less file if you wish.

Variables

Variables are defined starting with an @ symbol. For example:

@brand-primary: #0096db;

The @brand-primary variable has been set to color value #0096db. This variable can now be used multiple times by referencing the variable name when setting a CSS attribute, for example:

.section-heading {color: @brand-primary;}
.section-subheading {color: @brand-primary;}

The power of LESS allows you to do some rather cool stuff...like this:

.section-heading {color: darken(@brand-primary,20%);}
.section-subheading {color: lighten(@brand-primary,50%);}

Using the example above, you can lighten or darken colors and let LESS do the math for you!

Mixins

Mixins allow you to reuse code, and pass variables into the code. If you're familiar with other programming languages, it's a bit like using a JavaScript function/method.

For example, let's say we have multiple elements which need to use the background-size: cover attribute. We can do this:

.background-cover() {
    -webkit-background-size: cover;
    -moz-background-size: cover;
    background-size: cover;
    -o-background-size: cover;
}

The above code is an example of a mixin for the background-size: cover attribute. Now we can use it like this:

.bigimage {
    .background-cover;
}
.bigimage-two {
    .background-cover;
}

When the LESS file is compiled, it will look like this in our compiled CSS file:

.bigimage,.big-image-two {
    -webkit-background-size: cover;
    -moz-background-size: cover;
    background-size: cover;
    -o-background-size: cover;
}
Nesting

Another useful feature of LESS is nesting. Nesting can be used to increase legibility in your code and often reduces the amount of lines that need to be written. For example:

aside.clients img {
  margin: 50px auto;
}
aside.clients .carousel-indicators {
  position: static;
  margin: 0 auto 50px auto;
  bottom: 0;
  left: auto;
}
aside.clients .carousel-indicators li {
  background-color: #999;
  background-color: rgba(70, 70, 70, 0.25);
  border: 0;
  margin: 0 3px 0 3px;
  width: 10px;
  height: 10px;
}
aside.clients .carousel-indicators .active {
  background-color: #777;
  border: 0;
  width: 10px;
  height: 10px;
}

The above CSS can be written as below in LESS using nesting:

aside.clients {
    img {
        margin: 50px auto;
    }

    .carousel-indicators {
        position: static;
        margin: 0 auto 50px auto;
        bottom: 0;
        left: auto;
    }

    .carousel-indicators li {
      background-color: #999;
      background-color: rgba(70,70,70,.25);
      border: 0;
      margin: 0 3px 0 3px;
      width: 10px;
      height: 10px;
    }

    .carousel-indicators .active {
      background-color: #777;

Using SASS

We have also included SASS files, if LESS doesn't float your boat. SASS is the alternative of LESS, and it's growing ever more popular. You can use SASS in a very similar way to LESS, writing native CSS if you wish, but also writing specific SASS mixins, variables, etc.

SASS works in a very similar way to LESS, as it is also a CSS pre-processing language.

Import

CSS has an import option that lets you split your CSS into smaller, more maintainable portions. The only drawback is that each time you use @import in CSS it creates another HTTP request. Sass builds on top of the current CSS @import but instead of requiring an HTTP request, Sass will take the file that you want to import and combine it with the file you're importing into so you can serve a single CSS file to the web browser.

// _reset.scss

html,
body,
ul,
ol {
   margin: 0;
  padding: 0;
}
/* base.scss */

@import 'reset';

body {
  font: 100% Helvetica, sans-serif;
  background-color: #efefef;
}
Mixins

Some things in CSS are a bit tedious to write, especially with CSS3 and the many vendor prefixes that exist. A mixin lets you make groups of CSS declarations that you want to reuse throughout your site. You can even pass in values to make your mixin more flexible. A good use of a mixin is for vendor prefixes. Here's an example for border-radius.

@mixin border-radius($radius) {
  -webkit-border-radius: $radius;
     -moz-border-radius: $radius;
      -ms-border-radius: $radius;
          border-radius: $radius;
}

.box { @include border-radius(10px); }

Performance Tip

Some JavaScript plugins are rather heavy (they're big in file size) – the problem is that this will decrease the performance of your website. Try to use as few plugins as possible, using native JavaScript or CSS instead where possible. If you have to have many scripts, you can minify them, and also combine them into one file to improve performance.

You can also load jQuery from a CDN (such as Google Libraries) to improve performance.

Using CoffeeScript

CoffeeScript is a language which compiles to JavaScript. CoffeeScript is an attempt to improve the syntax of JavaScript, adopting some of the methods of syntactic sugar used by Ruby, Python and Haskell. CoffeeScript improves the brevity and readability of JavaScript code.

The best thing about CoffeeScript? You can write native JavaScript in a .coffee file - so you don't have to learn all the methods of CoffeeScript straight away!

Examples of CoffeeScript vs. JavaScript Code Segments

In JavaScript:
var mass = 72;

var BMI = mass / height / height;
if (18.5 < BMI && BMI < 25) alert('You are healthy!');
In CoffeeScript:
mass = 72
height = 1.78
BMI = mass / height**2
alert 'You are healthy!' if 18.5 < BMI < 25

Loading Screen

Browsers always display page content gradually, while it is loaded. This often results in pages which appear broken because some assets were not yet fully loaded, but the HTML markup and stylesheets are usually ready before webfonts, images and JavaScript. Slow loading is often due to Internet connection speeds, so it is hard to have full control of the page loading speed.

The loading screen is a nice, simple way of resolving this problem. An overlay, containing friendly text informing the user of what is happening, displays while all the assets are loading in the background. When the window is ready (all assets loaded), the loading screen is hidden with a fadeOut effect.

HTML

<div class="loading-screen">
    <div class="content animated fadeInDown">
        <i class="fa fa-circle-o-notch fa-spin"></i>
        <h2 class="section-heading">Baking the page</h2>
        <h3 class="section-subheading">Please wait...</h3>
    </div>
</div>

CSS Animation

CSS animation is a powerful way of adding animation to your website. It is supported in most browers, it's fast, and it's easy to implement.

All our themes are bundled with Animate.css, a collection of CSS animations combined into a single stylesheet. This allows us to bring a wide range of CSS animations to your fingertips.

Dynamically add animations using jQuery with ease:

$('#yourElement').addClass('animated bounceOutLeft');

You can also detect when an animation ends:

$('#yourElement').one('webkitAnimationEnd mozAnimationEnd MSAnimationEnd oanimationend animationend', doSomething);

Changing animation duration

You can change the duration of your animations, add a delay or change the number of times that it plays:

#yourElement {
  -vendor-animation-duration: 3s;
  -vendor-animation-delay: 2s;
  -vendor-animation-iteration-count: infinite;
}

PHP

The booking form is submitted to a PHP script which handles the data submitted and prepares a notification email using the PHP mail() function. The script file is located at assets/php/mail/booking.php

Change the $to variable to an email address of your choice. This address will receive a notification each time the form is completed.

The $email_subject variable is used for the email Subject line.

The $email_body variable can be altered to change the formatting and content of the notification email.

You can add a message textarea to your form, but you will need to uncomment the references to $_POST['message'] and $message for this to work.

Getting stuck with the PHP script? Email support@jcthemes.com for assistance. If you are receiving PHP errors, please revert your code to the original code supplied with the theme before it was edited/changed and try again. We cannot support any custom PHP added to the script.

<?php

// Check for empty fields

if(empty($_POST['name']) ||
   empty($_POST['email']) ||
   empty($_POST['phone']) ||
   //empty($_POST['message'])        ||
   empty($_POST['reservDate']) ||
   empty($_POST['numGuests']) ||
   empty($_POST['altDate']) ||
   empty($_POST['bookingTime']) ||
   !filter_var($_POST['email'],FILTER_VALIDATE_EMAIL))
   {
    echo "No arguments Provided!";
    return false;
   }
    
   $name = $_POST['name'];
   $email_address = $_POST['email'];
   $phone = $_POST['phone'];
   $reservDate = $_POST['reservDate'];
   $numGuests = $_POST['numGuests'];
   $altDate = $_POST['altDate'];
   $bookingTime = $_POST['bookingTime'];
   //$message          = $_POST['message'];

   // Create the email and send the message

   $to               = 'youremail@domain.com'; // Add your email address inbetween the '' replacing yourname@yourdomain.com - This is where the form will send a message to.
   $email_subject    = "Website Booking: $name";
   $email_body       = "You have received a new message from your website booking form.\n\n"."Here are the details:\n\nName: $name\n\nEmail: $email_address\n\nPhone: $phone\n\nReservation Date: $reservDate\n\nAlternative Date: $altDate\n\nNumber of Guests: $numGuests\n\nBooking Time: $bookingTime\n\n";
   $headers          = "From: noreply@yourdomain.com\n"; // This is the email address the generated message will be from. We recommend using something like noreply@yourdomain.com.
   $headers          .= "Reply-To: $email_address";

   mail($to,$email_subject,$email_body,$headers);

   return true;         
?>
